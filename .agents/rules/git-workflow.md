# Git Workflow

## 1. Commit Message Rules
- Use prefixes: `feat:`, `fix:`, `docs:`, `refactor:`, `chore:`.

## 2. Branching Strategy
- Maintain the `main` branch in a deployable state.
- Perform feature development in separate branches.
